## 1. Область действия

Настоящая Политика конфиденциальности описывает, какие персональные данные собирает и
обрабатывает мобильное приложение **[название приложения]** (далее — «Приложение»), с какой
целью, на каком правовом основании, кому передаются данные и какие права есть у пользователя
в соответствии с Общим регламентом по защите данных (GDPR, Регламент ЕС 2016/679) и
применимым законодательством Латвийской Республики.

## 2. Какие данные мы собираем

### 2.1 Данные аккаунта
При регистрации/входе (через Supabase Auth — email+пароль или вход через сторонний
аккаунт, если подключён):
- адрес электронной почты;
- идентификатор пользователя, сгенерированный системой;
- (при входе через соцсеть/Google) — публичное имя и аватар, если провайдер их передаёт.

### 2.2 Данные, создаваемые внутри приложения
- сохранённые списки покупок и их содержимое (названия товаров, категории, количество);
- избранные магазины/сети;
- история сравнений цен (для улучшения рекомендаций).

Эти данные являются пользовательским контентом и **не включают персональные данные третьих
лиц** — только сведения о покупках самого пользователя.

### 2.3 Технические данные и данные использования
- IP-адрес, тип устройства, версия ОС, версия приложения — стандартные технические логи;
- обезличенная аналитика использования приложения (какие экраны/функции используются) —
  если и когда будет подключена аналитика, это будет отражено здесь отдельным пунктом до
  включения такого сбора.

### 2.4 Данные для показа рекламы (Google AdMob)
Приложение показывает баннерную рекламу через Google AdMob. AdMob как самостоятельный
оператор обработки данных может собирать:
- рекламный идентификатор устройства (Advertising ID / IDFA для iOS, GAID для Android);
- данные о взаимодействии с рекламой (показы, клики) для оплаты и статистики;
- при наличии согласия пользователя на персонализированную рекламу — дополнительные данные
  для таргетинга.

Подробности о том, как Google обрабатывает эти данные, описаны в политике конфиденциальности
Google: https://policies.google.com/privacy. Приложение получает согласие пользователя на
персонализированную рекламу отдельно (см. §5 и `DATA_PROCESSING_CONSENT.md`) — без согласия
показывается только неперсонализированная реклама, где это технически предусмотрено AdMob.

### 2.5 Данные о ценах и товарах
Каталог товаров и цен формируется из **открытых, публично доступных источников** (страницы
интернет-магазинов сетей Rimi и Barbora — см. пояснение о происхождении цен ниже). Эти данные
не являются персональными данными пользователей Приложения и не связаны с личными покупками
конкретного человека в реальном магазине.

**Дисклеймер об источниках и точности цен:** цены и акции в Приложении собираются из открытых
источников (публичные страницы интернет-магазинов) на определённый момент времени и могут
отличаться от актуальной цены на кассе в конкретном физическом магазине. В частности, цены
по сети Maxima берутся с сайта интернет-магазина Barbora — сайта, который может показывать
цены, отличающиеся от цен в физических магазинах Maxima. Приложение не гарантирует полное
совпадение отображаемой цены с ценой в момент покупки.

## 3. Цели и правовые основания обработки

| Категория данных | Цель обработки | Правовое основание (GDPR) |
|---|---|---|
| Данные аккаунта | Регистрация, вход, идентификация пользователя | Исполнение договора (ст. 6(1)(b)) |
| Списки покупок, избранное | Основная функциональность приложения | Исполнение договора (ст. 6(1)(b)) |
| Технические логи | Безопасность, устранение неисправностей | Законный интерес (ст. 6(1)(f)) |
| Реклама (неперсонализированная) | Монетизация приложения | Законный интерес (ст. 6(1)(f)) |
| Реклама (персонализированная) | Монетизация, релевантная реклама | Согласие (ст. 6(1)(a)) |

## 4. Кому передаются данные (получатели / субобработчики)

- **Supabase** (база данных, аутентификация, хранение) — обработчик данных (data processor),
  данные размещаются в выбранном регионе дата-центра (рекомендация: выбрать регион в ЕС при
  настройке проекта Supabase, чтобы минимизировать трансграничную передачу данных).
- **Google (AdMob)** — независимый оператор для целей показа рекламы; возможна передача
  данных за пределы ЕЭЗ по стандартным договорным условиям Google (Standard Contractual
  Clauses).
- Мы **не продаём** персональные данные третьим лицам.

## 5. Согласие и его отзыв

Отдельное согласие запрашивается на:
- обработку персональных данных при регистрации аккаунта (см. `DATA_PROCESSING_CONSENT.md`);
- показ персонализированной рекламы (можно отключить в настройках приложения в любой момент
  без последствий для основной функциональности).

Пользователь может отозвать согласие в любой момент через настройки приложения или обратившись
на [email из шапки документа].

## 6. Сроки хранения данных

- Данные аккаунта и пользовательский контент (списки покупок, избранное) — хранятся, пока
  аккаунт активен; удаляются при удалении аккаунта (см. `ACCOUNT_DELETION_POLICY.md`).
- Технические логи — хранятся не дольше [например, 90 дней], затем удаляются или обезличиваются.
- Данные для рекламы — согласно политикам Google AdMob, вне прямого контроля Приложения.

## 7. Права пользователя

В соответствии с GDPR у пользователя есть право:
- на доступ к своим данным (ст. 15);
- на исправление неточных данных (ст. 16);
- на удаление данных / «право на забвение» (ст. 17) — см. `ACCOUNT_DELETION_POLICY.md`;
- на ограничение обработки (ст. 18);
- на переносимость данных (ст. 20);
- на возражение против обработки (ст. 21), в том числе против обработки для рекламы;
- отозвать согласие в любой момент без ущерба для законности предыдущей обработки (ст. 7(3));
- подать жалобу в надзорный орган — в Латвии это **Datu valsts inspekcija** (Data State
  Inspectorate), www.dvi.gov.lv.

Для реализации любого из этих прав — обращение на [email].

## 8. Дети

Приложение не предназначено для детей младше 16 лет и не собирает данные несовершеннолетних
целенаправленно. Если станет известно, что данные ребёнка были собраны без согласия законного
представителя, они будут удалены.

## 9. Безопасность

Применяются разумные технические и организационные меры защиты данных (шифрование при
передаче, ограничение доступа, аутентификация через Supabase Auth). Ни одна система не может
гарантировать абсолютную защиту, но мы стремимся минимизировать риски.

при следующем входе) до вступления изменений в силу.
