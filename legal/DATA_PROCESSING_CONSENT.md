# Согласие на обработку персональных данных (черновик v0.1)

> **Статус документа:** черновик текста согласия для экрана регистрации/входа и для настроек
> рекламы. Требует юридической проверки перед использованием в продакшене. Финальный текст
> должен быть на латышском (основной), плюс русский/английский — по языку интерфейса
> пользователя. Плейсхолдеры `[...]` заполнить перед публикацией.

## Назначение документа

Это не отдельная политика, а **конкретные формулировки согласия**, которые показываются
пользователю в интерфейсе приложения в двух местах:
1. При регистрации аккаунта — обязательное согласие для использования приложения.
2. В настройках рекламы — отдельное, необязательное согласие на персонализированную рекламу.

Оба согласия должны быть явными (checkbox не должен быть предзаполнен), запрашиваться
раздельно (нельзя объединять обязательное согласие на обработку данных аккаунта с
необязательным согласием на рекламу в один чекбокс — это нарушение принципа свободного и
раздельного согласия по GDPR).

---

## 1. Согласие при регистрации (обязательное для создания аккаунта)

**Текст на экране регистрации:**

> Регистрируясь, вы соглашаетесь с [Пользовательским соглашением] и подтверждаете, что
> ознакомились с [Политикой конфиденциальности]. Мы обрабатываем ваш email и данные вашего
> аккаунта (сохранённые списки покупок, избранные магазины) для предоставления функций
> приложения. Подробнее — в Политике конфиденциальности.
>
> ☐ Я согласен(-а) с условиями обработки данных, описанными в Политике конфиденциальности.

- Чекбокс **обязателен** — без него кнопка «Зарегистрироваться» неактивна.
- Ссылки на `[Пользовательское соглашение]` и `[Политика конфиденциальности]` должны открывать
  соответствующие документы внутри приложения (не только на внешнем сайте).
- Правовое основание: ст. 6(1)(b) GDPR (обработка необходима для исполнения договора —
  предоставления сервиса, который запросил пользователь) — формально это можно оформить и без
  отдельного «согласия», но явный чекбокс с подтверждением ознакомления повышает прозрачность
  и снижает юридические риски.

## 2. Согласие на персонализированную рекламу (необязательное, opt-in)

**Текст в настройках / при первом показе рекламы:**

> Мы показываем рекламу через Google AdMob, чтобы приложение оставалось бесплатным. Вы можете
> разрешить показ персонализированной рекламы (более релевантной вам) — для этого Google будет
> использовать рекламный идентификатор вашего устройства. Если вы откажетесь, будет
> показываться неперсонализированная реклама (если технически доступна для вашего региона), а
> основные функции приложения будут работать без изменений.
>
> ☐ Разрешить персонализированную рекламу

- По умолчанию — **выключено** (opt-in, не opt-out).
- Отдельная кнопка/переключатель, независимый от согласия из §1.
- Можно изменить в любой момент в настройках приложения.
- Правовое основание при согласии: ст. 6(1)(a) GDPR + требования ePrivacy Directive
  (использование рекламных идентификаторов приравнивается к использованию cookies на
  вебсайтах — нужен явный opt-in).
- **Технический момент для реализации:** на Android учитывать требования Google
  User Messaging Platform (UMP SDK) / Google's EU User Consent Policy — сама реализация
  экрана согласия для AdMob в ЕЭЗ должна соответствовать формату, который Google ожидает от
  приложений, показывающих рекламу европейским пользователям. Это отдельная техническая задача
  при интеграции AdMob, не только текстовая.

## 3. Право на отзыв согласия

В обоих случаях согласие можно отозвать:
- согласие на обработку данных аккаунта — через удаление аккаунта (см.
  `ACCOUNT_DELETION_POLICY.md`), так как без базовой обработки данных функциональность
  аккаунта невозможна;
- согласие на персонализированную рекламу — в любой момент через переключатель в настройках,
  без удаления аккаунта.

## 4. Журналирование согласий

Рекомендация для разработки: хранить в БД факт и время получения каждого согласия
(`consent_type`, `granted_at`, `revoked_at`, версия текста согласия, на который согласился
пользователь) — это нужно, чтобы иметь возможность доказать соответствие GDPR при проверке.
